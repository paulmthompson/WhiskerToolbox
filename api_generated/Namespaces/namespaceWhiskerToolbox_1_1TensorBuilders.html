Build a ColumnProviderFn that extracts a property (start, end, or duration) from the row intervals.

Parameters:

  • intervals Row intervals
  • property Which property to extract

Return: ColumnProviderFn producing one float per interval

This does NOT consult any DataManager source — the values come entirely from the interval endpoints.

function buildAnalogSampleAtOffsetProvider

ColumnProviderFn buildAnalogSampleAtOffsetProvider(
    DataManager & dm,
    std::string const & source_key,
    std::vector< TimeFrameIndex > const & row_times,
    int64_t offset
)

Build a ColumnProviderFn that samples an AnalogTimeSeries at each row timestamp plus a fixed offset.

Parameters:

  • dm DataManager that owns the source data
  • source_key DataManager key for an AnalogTimeSeries
  • row_times Ordered vector of TimeFrameIndex values (one per row)
  • offset Time offset to add to each row timestamp (may be negative)

Exceptions:

  • std::runtime_error if source_key is not an AnalogTimeSeries

Return: ColumnProviderFn producing one float per row

For each row i the provider reads source.getAtTime(row_times[i] + offset). Returns NaN for any sample that falls outside the source range.

This replaces the old AnalogTimestampOffsetsMultiComputer — each offset produces one call to this function, yielding one column.

function buildPipelineColumnProvider

ColumnProviderFn buildPipelineColumnProvider(
    DataManager & dm,
    std::string const & source_key,
    std::vector< TimeFrameIndex > const & row_times,
    WhiskerToolbox::Transforms::V2::TransformPipeline pipeline
)

Build a generic ColumnProviderFn for timestamp-row tensors from any source type.

Parameters:

  • dm DataManager that owns the source data
  • source_key DataManager key for any supported source type
  • row_times Ordered vector of TimeFrameIndex values (one per row)
  • pipeline TransformPipeline (may be empty for passthrough)

Exceptions:

  • std::runtime_error if the pipeline does not produce float output, has a terminal range reduction, or the source is unavailable

See: buildIntervalPipelineProvider() for interval-row tensors with reductions

Return: ColumnProviderFn producing one float per row

Works with any DataManager source type (AnalogTimeSeries, MaskData, LineData, PointData, etc.) as long as the pipeline transforms the source into float output (AnalogTimeSeries or RaggedAnalogTimeSeries).

Empty pipeline (passthrough): The source must already be a float type (AnalogTimeSeries or RaggedAnalogTimeSeries). Samples the source directly at each row timestamp.

Non-empty pipeline: Calls executePipeline() on the full source DataTypeVariant, then samples the resulting AnalogTimeSeries or RaggedAnalogTimeSeries at each row timestamp.

Terminal range reductions are rejected — they collapse the entire series to a single scalar, which is not meaningful for timestamp-row columns. For interval-row columns that need range reductions, use buildIntervalPipelineProvider() instead.

Validation is performed at build time via pipelineProducesFloat().

function buildIntervalPipelineProvider

ColumnProviderFn buildIntervalPipelineProvider(
    DataManager & dm,
    std::string const & source_key,
    std::shared_ptr< DigitalIntervalSeries > intervals,
    WhiskerToolbox::Transforms::V2::TransformPipeline pipeline
)

Build a generic ColumnProviderFn for interval-row tensors from any source type.

Parameters:

  • dm DataManager that owns the source data
  • source_key DataManager key for any supported source type
  • intervals Shared pointer to the DigitalIntervalSeries defining rows
  • pipeline TransformPipeline that must end with a range reduction

Exceptions:

  • std::runtime_error if the pipeline does not produce float output, lacks a range reduction, or the source is unavailable

See: buildPipelineColumnProvider() for timestamp-row tensors without reductions

Return: ColumnProviderFn producing one float per interval

Note: RaggedAnalogTimeSeries sources are not supported (no GatherResult specialisation exists). Passing one will throw at materialization time.

Works with any DataManager source type that satisfies HasDataTraits (AnalogTimeSeries, MaskData, LineData, PointData, DigitalEventSeries, DigitalIntervalSeries) by delegating to gatherAndExecutePipeline().

For each interval row, the function:

  1. Gathers a view/copy of the source data within the interval boundaries
  2. Executes the pipeline on that gathered view
  3. Extracts a single float from the pipeline output via extractSingleFloat()

The pipeline must contain a range reduction that collapses each gathered view to a single scalar. Pipelines without a range reduction are rejected at build time.

Validation is performed at build time via pipelineProducesFloat().

function buildProviderFromRecipe

ColumnProviderFn buildProviderFromRecipe(
    DataManager & dm,
    ColumnRecipe const & recipe,
    std::vector< TimeFrameIndex > const & row_times,
    std::shared_ptr< DigitalIntervalSeries > intervals
)

Build a ColumnProviderFn from a ColumnRecipe.

Parameters:

  • dm DataManager that owns source data
  • recipe Column specification
  • row_times Row timestamps (for timestamp-row tensors; empty for interval-row)
  • intervals Row intervals (for interval-row tensors; nullptr for timestamp-row)

Return: ColumnProviderFn

This is the main entry point for building providers from serialized configurations. It dispatches to the appropriate builder based on the recipe’s fields.

function buildInvalidationWiringFn

InvalidationWiringFn buildInvalidationWiringFn(
    DataManager & dm,
    std::vector< std::string > const & source_keys
)

Build an InvalidationWiringFn that registers DataManager observers to invalidate tensor columns when source data changes.

Parameters:

  • dm DataManager that owns source data
  • source_keys Ordered list of DataManager keys, one per column. Empty strings mean “no source dependency” (e.g. interval property columns) and will be skipped.

Return: InvalidationWiringFn suitable for TensorData::createFromLazyColumns()

For each (column_index, source_key) pair, the wiring function registers an observer on the source data. When the source notifies, the wiring callback invalidates the corresponding column and notifies the tensor’s own observers.
