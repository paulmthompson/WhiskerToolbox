using WhiskerToolbox::Gather::ViewAdaptorFn = typedef std::function<std::vector<OutElement>(std::span<InElement const>)>;

Type-erased view adaptor function.

Note: This is primarily used when type erasure is needed (e.g., storing in containers, passing through non-template interfaces).

Takes a span of input elements and produces a vector of output elements. This is the materialized version - for lazy evaluation, use the templated view adaptor that returns a range adaptor.

  • pre_window Absolute time units before alignment point
  • post_window Absolute time units after alignment point

The window is specified in absolute time units. The adapter produces intervals centered on the alignment point: [align - pre, align + post].

function extractSingleFloat

float extractSingleFloat(
    DataTypeVariant const & output
)

Extract a single float from a DataTypeVariant pipeline output.

Parameters:

  • output Pipeline output wrapped in a DataTypeVariant

Exceptions:

  • std::runtime_error if the variant does not hold a float-producing type

Return: float scalar extracted from the output

Called after executing a range-reduction pipeline on one gathered interval view. The pipeline’s output should be a single-element AnalogTimeSeries (when the reduction produces a scalar) or a RaggedAnalogTimeSeries (for ragged intermediate results).

  • AnalogTimeSeries → returns values[0], or NAN if empty
  • RaggedAnalogTimeSeries → returns first value at the first time point, or NAN if empty
  • Any other type → throws std::runtime_error

function gatherAndExecutePipeline

std::vector< float > gatherAndExecutePipeline(
    DataTypeVariant const & source,
    std::shared_ptr< DigitalIntervalSeries > intervals,
    WhiskerToolbox::Transforms::V2::TransformPipeline const & pipeline
)

Gather source data over intervals and execute a pipeline on each gathered view, returning one float per interval.

Parameters:

  • source Source data wrapped in a DataTypeVariant
  • intervals DigitalIntervalSeries defining the row intervals
  • pipeline TransformPipeline to execute on each gathered view. Should end with a range reduction that collapses the gathered data to a single scalar.

Exceptions:

  • std::runtime_error if the source type does not satisfy HasDataTraits or if extractSingleFloat() fails on the pipeline output

Return: std::vector with one element per interval

For each interval i the function:

  1. Creates a GatherResult view/copy of the source data within [intervals[i].start, intervals[i].end].
  2. Executes pipeline on that gathered view (T → DataTypeVariant).
  3. Calls extractSingleFloat() on the result.

If the source data and intervals use different TimeFrames, the interval boundaries are converted to the source’s coordinate system before gathering (same logic as TensorColumnBuilders::prepareIntervalsForGather).

Dispatch is fully type-erased: the function visits the DataTypeVariant and uses GatherResult::create() + pipeline.execute() for each concrete contained type that satisfies TypeTraits::HasDataTraits.
