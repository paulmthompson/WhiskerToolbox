Types Documentation

enum ScalingMode

Enumerator Value Description
FiringRateHz count / (sample_spacing_s × num_trials) → spikes/s
ZScore Per-unit z-score: (rate - mean) / std across all time points.
Normalized01 Per-unit min-max normalization to [0, 1].
RawCount Raw event count (no normalization)
CountPerTrial count / num_trials

Scaling mode for rate estimate values.

Controls how raw event counts are transformed. Shared between HeatmapWidget, PSTHWidget, and any future rate-based plot widgets. Applied per-unit (per-row for heatmaps) after rate estimation.

using EstimationParams

using WhiskerToolbox::Plots::EstimationParams = typedef std::variant<BinningParams, GaussianKernelParams, CausalExponentialParams>;

Tagged union of all supported rate estimation method parameter structs.

Note: GaussianKernelParams and CausalExponentialParams are included in the variant but their implementations return empty RateEstimate (stub).

This is the single variation point for estimation method selection. Selecting a method at runtime is zero-overhead compared to a virtual base-class strategy — std::visit dispatches via a jump table.

To add a new method:

  1. Define a new *Params struct above
  2. Append it to this alias
  3. Add the implementation in [EventRateEstimation.cpp](/api_generated/Files/EventRateEstimation_8cpp.md#file-eventrateestimation.cpp)

using HeatmapScaling

using WhiskerToolbox::Plots::HeatmapScaling = typedef ScalingMode;

Use ScalingMode instead

using TooltipHitTestFn

using WhiskerToolbox::Plots::TooltipHitTestFn = typedef std::function<std::optional<PlotTooltipHit>(QPoint screen_pos)>;

Callback that performs a hit test at a screen position.

Returns a PlotTooltipHit on success, or std::nullopt if nothing is under the cursor. This callback may perform spatial queries (QuadTree, etc.) and is only called after the dwell timer fires.

using TooltipTextFn

using TooltipContentFn

using WhiskerToolbox::Plots::TooltipContentFn = typedef std::function<PlotTooltipContent(PlotTooltipHit const & hit)>;

Callback that generates rich tooltip content from a hit result.

Use this for expensive content generation (rendering glyphs, mini-plots). Returns a full PlotTooltipContent, allowing the provider to choose between text and pixmap on a per-hit basis.

Create a UnitGatherContext for a single DigitalEventSeries key.

Parameters:

  • data_manager DataManager containing the event and alignment series
  • event_key DataManager key of the [DigitalEventSeries](/api_generated/Classes/classDigitalEventSeries.md) to gather
  • alignment_data Alignment configuration (alignment key, window size, etc.)

  • The gather result is empty (alignment key not found or no events in window)
  • The source series does not have a [TimeFrame](/api_generated/Classes/classTimeFrame.md)

function createUnitGatherContexts

std::vector< UnitGatherContext > createUnitGatherContexts(
    std::shared_ptr< DataManager > const & data_manager,
    std::vector< std::string > const & event_keys,
    PlotAlignmentData const & alignment_data
)

Create UnitGatherContexts for a batch of DigitalEventSeries keys.

Parameters:

  • data_manager DataManager containing the event and alignment series
  • event_keys DataManager keys of the [DigitalEventSeries](/api_generated/Classes/classDigitalEventSeries.md) to gather
  • alignment_data Alignment configuration shared across all units

Return: Vector of successfully gathered [UnitGatherContext](/api_generated/Classes/structWhiskerToolbox_1_1Plots_1_1UnitGatherContext.md) objects

Applies createUnitGatherContext to each key in event_keys with the same alignment settings. Keys for which gathering fails are silently skipped; the returned vector may therefore be shorter than event_keys.

Order is preserved: the i-th element of the result corresponds to the i-th key that succeeded.

function estimateRate

RateEstimate estimateRate(
    GatherResult< DigitalEventSeries > const & gathered,
    TimeFrame const * time_frame,
    double window_size,
    EstimationParams const & params =BinningParams{}
)

Estimate the rate for a single unit (aggregate across trials)

Parameters:

  • gathered Trial-aligned views of one [DigitalEventSeries](/api_generated/Classes/classDigitalEventSeries.md)
  • time_frame TimeFrame of the source series (may be null)
  • window_size Total window span in time units (bins cover ±window_size/2)
  • params Estimation method and parameters (default: [BinningParams](/api_generated/Classes/structWhiskerToolbox_1_1Plots_1_1BinningParams.md){})

Return: [RateEstimate](/api_generated/Classes/structWhiskerToolbox_1_1Plots_1_1RateEstimate.md) with times, values, num_trials, and metadata

Returns a [RateEstimate](/api_generated/Classes/structWhiskerToolbox_1_1Plots_1_1RateEstimate.md) with paired times[] (bin centers for binning) and values[] (raw event counts summed across all trials).

Uses time_frame to convert each event’s [TimeFrameIndex](/api_generated/Classes/structTimeFrameIndex.md) to an absolute time before computing the relative time with respect to the per-trial alignment time from gathered.

If time_frame is null, [TimeFrameIndex::getValue()](/api_generated/Classes/structTimeFrameIndex.md#function-getvalue) is used directly as the absolute time (identity mapping).

function estimateRateWithTrials

RateEstimateWithTrials estimateRateWithTrials(
    GatherResult< DigitalEventSeries > const & gathered,
    TimeFrame const * time_frame,
    double window_size,
    EstimationParams const & params =BinningParams{}
)

Estimate rate with per-trial breakdown for a single unit.

Parameters:

  • gathered Trial-aligned views of one [DigitalEventSeries](/api_generated/Classes/classDigitalEventSeries.md)
  • time_frame TimeFrame of the source series (may be null)
  • window_size Total window span in time units
  • params Estimation method and parameters (default: [BinningParams](/api_generated/Classes/structWhiskerToolbox_1_1Plots_1_1BinningParams.md){})

Return: [RateEstimateWithTrials](/api_generated/Classes/structWhiskerToolbox_1_1Plots_1_1RateEstimateWithTrials.md) with aggregate + per-trial data

Same as [estimateRate()](/api_generated/Namespaces/namespaceWhiskerToolbox_1_1Plots.md#function-estimaterate) but additionally returns per-trial rate curves in [RateEstimateWithTrials::trials](/api_generated/Classes/structWhiskerToolbox_1_1Plots_1_1RateEstimateWithTrials.md#variable-trials). The aggregate estimate is the trial-summed result (same as [estimateRate()](/api_generated/Namespaces/namespaceWhiskerToolbox_1_1Plots.md#function-estimaterate)), and each trial’s individual histogram is stored separately.

Use this when you need uncertainty quantification (SEM, CI, bootstrap) via the functions in [RateUncertainty.hpp](/api_generated/Files/RateUncertainty_8hpp.md#file-rateuncertainty.hpp).

function estimateRates

std::vector< RateEstimate > estimateRates(

Estimate rates for multiple units.

function applyScaling

void applyScaling(
    RateEstimate & estimate,
    ScalingMode mode,
    double time_units_per_second
)

Apply a named scaling to a RateEstimate in-place.

Parameters:

  • estimate RateEstimate to modify in-place
  • mode Scaling mode to apply
  • time_units_per_second Conversion factor (e.g. 1000.0 for ms)

Dispatches to the appropriate normalization primitive based on mode. Reads estimate.metadata.sample_spacing and estimate.num_trials automatically.

ScalingMode Transform
FiringRateHz count / (sample_spacing_s × num_trials)
CountPerTrial count / num_trials
RawCount no transform
Normalized01 per-unit (v - min) / (max - min)
ZScore per-unit (v - mean) / std (after count/trial)

function pruneOverlappingAlignmentTimes

inline std::vector< size_t > pruneOverlappingAlignmentTimes(
    std::vector< int64_t > const & alignment_times,
    int64_t pre_window,
    int64_t post_window
)

Prune alignment times whose expanded windows would overlap.

Parameters:

  • alignment_times Sorted absolute alignment times
  • pre_window Time units before each alignment time
  • post_window Time units after each alignment time

Return: Indices into alignment_times that survive pruning

Precondition: alignment_times is sorted in ascending order

Uses a greedy left-to-right scan: keeps the first alignment time, then for each subsequent time checks whether its window [time - pre_window, time + post_window] overlaps with the window of the last kept time. If it overlaps, the event is skipped; otherwise it is kept.

Alignment times must be sorted in ascending order.
