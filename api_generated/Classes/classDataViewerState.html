Detailed Description

class DataViewerState;

State class for DataViewer_Widget.

DataViewerState is the Qt wrapper around DataViewerStateData that provides typed accessors and Qt signals for all state properties.

Signal Categories

  • View State: viewStateChanged
  • Theme: themeChanged
  • Grid: gridChanged
  • UI Preferences: uiPreferencesChanged
  • Interaction: interactionModeChanged
  • Series Options: Forwarded from SeriesOptionsRegistry

Thread Safety

This class is NOT thread-safe. All access should be from the main/GUI thread.

Public Signals Documentation

signal viewStateChanged

void viewStateChanged()

Emitted when any view state property changes.

This includes time window, Y bounds, zoom, pan, and spacing changes. For Y-axis-only changes (pan, zoom, bounds), the scene does NOT need rebuilding — only the projection matrix needs updating.

signal timeWindowChanged

void timeWindowChanged()

Emitted when the time window (x_min/x_max) changes.

This signal indicates that the data range has changed and buffers need rebuilding. Connect to this for scene-dirty operations. Always emitted together with viewStateChanged().

function seriesOptions

inline SeriesOptionsRegistry & seriesOptions()

Get the series options registry for generic access.

Return: Reference to SeriesOptionsRegistry

The registry provides a unified API for all series option types:


// Set options (type inferred)
AnalogSeriesOptionsData opts;
opts.hex_color() = "#ff0000";
state->seriesOptions().set("channel_1", opts);

// Get options (type explicit)
auto* opts = state->seriesOptions().get<AnalogSeriesOptionsData>("channel_1");

// Get visible keys
QStringList visible = state->seriesOptions().visibleKeys<AnalogSeriesOptionsData>();
