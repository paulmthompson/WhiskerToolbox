function loadSpikeToAnalogPairing

void loadSpikeToAnalogPairing(
    std::string const & digital_group,
    std::string const & analog_group,
    std::vector< SpikeToAnalogPairing > const & pairings,
    SpikeToAnalogPlacementMode mode,
    SpikeToAnalogParseConfig const & config =SpikeToAnalogParseConfig{},
    bool overlay_render_digital_events_as_boxes =false
)

Apply lane placement overrides based on spike-to-analog pairings.

Parameters:

  • digital_group Group prefix for digital (spike) series (e.g. “spikes”)
  • analog_group Group prefix for analog (LFP) series (e.g. “voltage”)
  • pairings Parsed pairings from parseSpikeToAnalogCSV
  • mode Lane placement policy
  • config Parse config; the digital_key_one_based and analog_key_one_based fields control how the numeric suffix in series key names is interpreted when building the channel-to-key lookup maps.
  • overlay_render_digital_events_as_boxes If true and mode is Overlay, set each affected digital event series’ glyph_shape to Box in state.

Precondition: pairings must not be empty; zero-length pairings are no-ops.

For each pairing in pairings, locates the digital series key (digital_group + “” + (digital_channel + 1)) and the analog series key (analog_group + ”” + (analog_channel + 1)) that are currently registered in the data store.

The placement policy applied is controlled by mode:

  • AdjacentBelow: the digital series is inserted in its own auto-lane immediately below the analog series’ lane.
  • AdjacentAbove: the digital series is inserted immediately above.
  • Overlay: the digital series shares the analog series’ lane_id (with overlay_mode set to Overlay).

The current visual order of all displayed series is read from _multi_lane_axis_state->lanes() to determine insertion positions. New lane_order values are assigned with the same ``(N - i) * 10 formula used throughout the ordering system.
