Detailed Description

class CorePlotting::Interaction::RectangleInteractionController;

Controller for interactive rectangle creation and modification.

See: IGlyphInteractionController for the interface contract

Supports two modes:

Creation Mode (via [start()](/api_generated/Classes/classCorePlotting_1_1Interaction_1_1RectangleInteractionController.md#function-start)):

  • User clicks to set first corner
  • Drags to set opposite corner
  • Rectangle preview updates during drag
  • Complete on mouse release

Modification Mode (via [startEdgeDrag()](/api_generated/Classes/classCorePlotting_1_1Interaction_1_1RectangleInteractionController.md#function-startedgedrag)):

  • Used when user clicks on an edge of an existing rectangle
  • Only the specified edge moves during drag
  • Shows ghost of original position

Interval Mode (constrain_to_x_axis = true):

  • Rectangle height spans full canvas
  • Only X coordinates matter (for DigitalIntervalSeries)

Selection Box Mode (constrain_to_x_axis = false):

  • User defines both width and height
  • For rectangular selection regions

function startEdgeDrag

void startEdgeDrag(
    float canvas_x,
    float canvas_y,
    std::string series_key,
    EntityId entity_id,
    RectangleEdge edge,
    glm::vec4 original_bounds
)

Start edge drag mode for modifying an existing rectangle.

Parameters:

  • canvas_x Starting X coordinate in canvas pixels
  • canvas_y Starting Y coordinate in canvas pixels
  • series_key Identifier of the series containing the rectangle
  • entity_id EntityId of the rectangle being modified
  • edge Which edge is being dragged
  • original_bounds Original rectangle {x, y, width, height} in canvas coords

Called when the user clicks on an edge of an existing rectangle. The widget should pre-process the hit test result and provide:

  • Which edge was hit
  • The original bounds of the rectangle
  • The EntityId of the rectangle being modified
