Detailed Description

class MLCore::MLModelOperation;

Abstract base for all ML model operations.

A model operation encapsulates a single ML algorithm. It can be trained (supervised) or fit (unsupervised), then used for prediction or cluster assignment. Each operation declares its task type so the registry can categorize it.

Supervised workflow

op->train(features, labels, params);
op->predict(features, predictions);
// Optional:
op->predictProbabilities(features, probabilities);

Unsupervised workflow

op->fit(features, params);
op->assignClusters(features, assignments);

Lifecycle

  1. Construct via factory (MLModelRegistry::create)
  2. Train/fit with parameters
  3. Query [isTrained()](/api_generated/Classes/classMLCore_1_1MLModelOperation.md#function-istrained) to confirm success
  4. Predict/assign on new data
  5. Optionally save/load for persistence

function train

inline virtual bool train(
    arma::mat const & features,
    arma::Row< std::size_t > const & labels,
    MLModelParametersBase const * params
)

Train the model on labeled data.

Parameters:

  • features Feature matrix (features × observations). Each column is one observation with features.n_rows feature values.
  • labels Label row vector (1 × observations). Each element is a class index in [0, num_classes).
  • params Algorithm-specific parameters. May be nullptr to use defaults. Must be castable to the concrete parameter type expected by this model.

Return: true if training succeeded

Precondition:

  • features must not be empty (enforcement: runtime_check)
  • features must not contain NaN or Inf values (enforcement: none) [IMPORTANT]
  • labels.n_elem must equal features.n_cols (enforcement: none) [IMPORTANT]
  • Features are expected to be z-score normalized for scale-sensitive classifiers (Logistic Regression, Softmax Regression). Tree-based methods (Random Forest) are scale-invariant. (enforcement: none) [IMPORTANT]

Reimplemented by: MLCore::MLSupervisedDimReductionOperation::train, MLCore::HiddenMarkovModelOperation::train, MLCore::LogisticRegressionOperation::train, MLCore::NaiveBayesOperation::train, MLCore::RandomForestOperation::train, MLCore::SoftmaxRegressionOperation::train

After successful training, [isTrained()](/api_generated/Classes/classMLCore_1_1MLModelOperation.md#function-istrained) returns true, and [numClasses()](/api_generated/Classes/classMLCore_1_1MLModelOperation.md#function-numclasses) and [numFeatures()](/api_generated/Classes/classMLCore_1_1MLModelOperation.md#function-numfeatures) reflect the training data dimensions.

The default implementation returns false (unsupervised models).

function trainSequences

inline virtual bool trainSequences(
    std::vector< arma::mat > const & featureSequences,
    std::vector< arma::Row< std::size_t > > const & labelSequences,
    MLModelParametersBase const * params
)

Train the model on pre-segmented temporal sequences.

Parameters:

  • featureSequences Vector of feature matrices, one per contiguous temporal segment. Each matrix is (features × segment_length).
  • labelSequences Vector of label row vectors, aligned 1:1 with featureSequences. Each row is (1 × segment_length).
  • params Algorithm-specific parameters. May be nullptr.

Return: true if training succeeded

Reimplemented by: MLCore::HiddenMarkovModelOperation::trainSequences

The default implementation concatenates all sequences into a single matrix and delegates to [train()](/api_generated/Classes/classMLCore_1_1MLModelOperation.md#function-train). Sequence-aware models (e.g., HMM) override this to preserve temporal structure.

inline virtual bool predictSequences(

)

Predict labels for pre-segmented temporal sequences.

Parameters:

  • featureSequences Vector of feature matrices, one per segment
  • predictionSequences [out] Vector of prediction row vectors, one per segment, aligned with featureSequences

Return: true if prediction succeeded

Reimplemented by: MLCore::HiddenMarkovModelOperation::predictSequences

The default implementation concatenates, calls [predict()](/api_generated/Classes/classMLCore_1_1MLModelOperation.md#function-predict), and splits the result back. Sequence-aware models override to preserve temporal continuity (e.g., Viterbi decoding per sequence).

function predictSequencesConstrained

inline virtual bool predictSequencesConstrained(
    std::vector< arma::mat > const & featureSequences,
    std::vector< arma::Row< std::size_t > > & predictionSequences,
    std::vector< std::optional< std::size_t > > const & initial_state_constraints
)

Predict labels with per-sequence initial state constraints.

Parameters:

  • featureSequences Vector of feature matrices, one per segment
  • predictionSequences [out] Predicted labels per segment
  • initial_state_constraints Per-sequence initial state override. If element i has a value, the model’s initial state distribution is clamped to a delta on that state for sequence i. If nullopt, the model’s learned initial distribution is used.

Return: true if prediction succeeded

Reimplemented by: MLCore::HiddenMarkovModelOperation::predictSequencesConstrained

The default implementation ignores constraints and delegates to predictSequences(). Sequence-aware models (e.g., HMM) override this to apply constrained Viterbi decoding.

function predictSequencesBidirectionalConstrained

inline virtual bool predictSequencesBidirectionalConstrained(
    std::vector< arma::mat > const & featureSequences,
    std::vector< arma::Row< std::size_t > > & predictionSequences,
    std::vector< std::optional< std::size_t > > const & initial_state_constraints,
    std::vector< std::optional< std::size_t > > const & terminal_state_constraints
)

Predict labels with both initial and terminal state constraints.

Parameters:

  • featureSequences Vector of feature matrices, one per segment
  • predictionSequences [out] Predicted labels per segment
  • initial_state_constraints Per-sequence initial state override (nullopt = unconstrained)
  • terminal_state_constraints Per-sequence terminal state override (nullopt = unconstrained)

Return: true if prediction succeeded

Reimplemented by: MLCore::HiddenMarkovModelOperation::predictSequencesBidirectionalConstrained

For sequence models (e.g., HMM), constraining both ends of a Viterbi path produces tighter predictions in unlabeled gaps between labeled blocks. The terminal constraint changes the backtrace start from argmax to a forced state.

The default implementation ignores terminal constraints and delegates to predictSequencesConstrained().

inline virtual bool predictProbabilitiesPerSequence(
    std::vector< arma::mat > const & featureSequences,
    std::vector< arma::mat > & probabilitySequences,
    std::vector< std::optional< std::size_t > > const & initial_state_constraints,
    std::vector< std::optional< std::size_t > > const & terminal_state_constraints
)

Compute per-sequence state probabilities with boundary constraints.

Parameters:

  • featureSequences Vector of feature matrices, one per segment
  • probabilitySequences [out] Vector of probability matrices (num_states × segment_length), one per segment
  • initial_state_constraints Per-sequence initial state override (nullopt = unconstrained)
  • terminal_state_constraints Per-sequence terminal state override (nullopt = unconstrained)

Return: true if estimation succeeded

Reimplemented by: MLCore::HiddenMarkovModelOperation::predictProbabilitiesPerSequence

For sequence models, runs Forward-Backward per segment with clamped initial and terminal state distributions.

The default implementation calls predictProbabilities() on each segment independently without constraints.

function fit

inline virtual bool fit(
    arma::mat const & features,
    MLModelParametersBase const * params
)

Fit the model to unlabeled data (unsupervised)

Parameters:

  • features Feature matrix (features × observations)
  • params Algorithm-specific parameters (K, epsilon, etc.)

Return: true if fitting succeeded

Precondition:

  • features must not be empty (enforcement: runtime_check)
  • features must not contain NaN or Inf values (enforcement: none) [IMPORTANT]
  • Features are expected to be z-score normalized for distance-based clustering algorithms (K-Means, DBSCAN, GMM). (enforcement: none) [IMPORTANT]

Reimplemented by: MLCore::MLDimReductionOperation::fit, MLCore::DBSCANOperation::fit, MLCore::GaussianMixtureOperation::fit, MLCore::KMeansOperation::fit

After successful fitting, [isTrained()](/api_generated/Classes/classMLCore_1_1MLModelOperation.md#function-istrained) returns true (the model is ready for [assignClusters()](/api_generated/Classes/classMLCore_1_1MLModelOperation.md#function-assignclusters)), and [numClasses()](/api_generated/Classes/classMLCore_1_1MLModelOperation.md#function-numclasses) returns the number of clusters found/requested.

The default implementation returns false (supervised models).
