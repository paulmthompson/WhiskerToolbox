Detailed Description

class WhiskerToolbox::Transforms::V2::TransformPipeline;

Builder for chaining multiple transforms together.

This class enables composing a sequence of transforms that will be applied in order, with automatic container type transitions.

The pipeline is fully generic and works with any compatible transforms.

Example usage: ```cpp

auto pipeline = TransformPipeline() .addStep(“CalculateMaskArea”, MaskAreaParams{}) .addStep(“SumReduction”, SumReductionParams{});

// Execute with correct input/output types std::shared_ptr result = pipeline.execute<MaskData, AnalogTimeSeries>(mask_data);

function execute

template <typename InputContainer >
DataTypeVariant execute(
    InputContainer const & input
) const

Execute pipeline and return variant result (Generalist)

See:

  • executeOptimized For compile-time known types with automatic optimization
  • executeFused For high-performance pure element-wise pipelines

Determines output container type at runtime based on pipeline steps and input raggedness. This is the primary entry point for runtime-composed pipelines (e.g. from UI) where the output type is not known at compile time.

Uses a buffered execution model that handles transitions between element-wise and time-grouped transforms.

Execute pipeline with automatic fusion optimization (Dispatcher)

Parameters:

  • input Input data

Template Parameters:

  • InputContainer Input container type
  • OutputContainer Output container type

See:

  • executeFused The specialized implementation for pure element pipelines
  • execute The general implementation handling time-grouped transforms

Return: Output container

Analyzes the pipeline structure and chooses the optimal execution strategy:

  • Pure element chains: Fused single-pass execution (via executeFused)
  • Chains with time-grouped: Materialized intermediate execution (via execute)

Use this when the output type is known at compile time.

function executeFused

template <typename InputContainer ,
typename OutputContainer >
inline std::shared_ptr< OutputContainer > executeFused(
    InputContainer const & input
) const

Execute a multi-step element-level pipeline with full fusion (Specialist)

Parameters:

  • input Input data

Template Parameters:

  • InputContainer Input container type
  • OutputContainer Output container type

See:

  • executeOptimized Automatic selection of this method
  • executeAsView For lazy evaluation without materialization

Return: Output with composed transformation applied

For element→element→element pipelines (no time-grouped transforms), this composes all transforms into a single function and executes in one pass with zero intermediate allocations.

Example: Mask2D → skeletonize → fitLine → resample Results in: for (mask : masks) { output.add(resample(fitLine(skeletonize(mask)))); }

Performance:

  • Single pass through input data
  • No intermediate container allocations
  • Intermediate values stay hot in CPU cache
  • Type erasure overhead (~1-2 pointer dereferences per element per step)

Implementation: Uses runtime function composition with type-erased intermediate values. This allows arbitrary pipeline lengths determined at runtime (e.g., from UI).
