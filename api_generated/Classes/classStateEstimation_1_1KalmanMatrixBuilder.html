Detailed Description

class StateEstimation::KalmanMatrixBuilder;

Utility class for building Kalman filter matrices for composite features.

This class constructs the state transition (F), measurement (H), process noise (Q), and measurement noise (R) matrices for tracking systems with heterogeneous features.

Features can have different temporal behaviors:

  • KINEMATIC_2D: 2D measurement (x,y) → 4D state (x, y, vx, vy)
  • STATIC: 1D measurement → 1D state (no velocity)
  • SCALAR_DYNAMIC: 1D measurement → 2D state (value + derivative)

The builder uses feature metadata to construct appropriate block-diagonal matrices where each block corresponds to one feature with its specific dynamics.

function buildF

static inline Eigen::MatrixXd buildF(
    std::vector< FeatureConfig > const & configs
)

Build F matrix (state transition) for N features.

Parameters:

  • configs Vector of feature configurations (one per feature)

Return: Block-diagonal F matrix (4N × 4N)

Each feature block is: [1 0 dt 0] [0 1 0 dt] [0 0 1 0] [0 0 0 1]

function buildH

static inline Eigen::MatrixXd buildH(
    int num_features
)

Build H matrix (measurement model) for N features.

Parameters:

  • num_features Number of features being tracked

Return: Block-diagonal H matrix (2N × 4N)

Each feature block is: [1 0 0 0] [0 1 0 0]

This extracts only the position components from the state.

function buildQ

static inline Eigen::MatrixXd buildQ(
    std::vector< FeatureConfig > const & configs
)

Build Q matrix (process noise covariance) for N features.

Parameters:

  • configs Vector of feature configurations (one per feature)

Return: Block-diagonal Q matrix (4N × 4N)

Each feature block is a 4×4 diagonal matrix: [σ_pos² 0 0 0 ] [0 σ_pos² 0 0 ] [0 0 σ_vel² 0 ] [0 0 0 σ_vel²]

Each feature block is a 2×2 diagonal matrix: [σ_meas² 0 ] [0 σ_meas²]

function buildFFromMetadata

static inline Eigen::MatrixXd buildFFromMetadata(
    std::vector< FeatureMetadata > const & metadata_list,
    FeatureConfig const & config
)

Build F matrix from feature metadata.

Parameters:

  • metadata_list Metadata for each feature in order
  • config Configuration (dt, noise parameters)

Return: Block-diagonal F matrix

Constructs a block-diagonal state transition matrix where each block corresponds to one feature’s temporal dynamics:

  • KINEMATIC_2D/3D: Position + velocity model with dt
  • STATIC: Identity (no change)
  • SCALAR_DYNAMIC: Value + derivative model with dt

function addCrossFeatureProcessNoise

static inline Eigen::MatrixXd addCrossFeatureProcessNoise(
    Eigen::MatrixXd Q,
    std::vector< FeatureMetadata > const & metadata_list,
    std::map< std::pair< int, int >, double > const & cross_correlations
)

Add cross-feature covariance to Q matrix.

Parameters:

  • Q The base Q matrix (typically block-diagonal)
  • metadata_list Metadata for each feature (to find state offsets)
  • cross_correlations Map of (feature_i, feature_j) -> correlation coefficient

Return: Modified Q matrix with cross-feature covariance

Modifies a process noise covariance matrix to include off-diagonal terms representing correlated process noise between features. This is useful when features are known to covary, such as position and measured length when camera clipping causes the measured length to depend on position.
