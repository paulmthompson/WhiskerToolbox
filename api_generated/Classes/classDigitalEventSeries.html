Detailed Description

class DigitalEventSeries;

A sorted collection of discrete time events with entity tracking.

DigitalEventSeries stores events as (TimeFrameIndex, EntityId) pairs, maintaining events in sorted order by time. Each event represents a discrete occurrence at a specific time point (compare to DigitalIntervalSeries for time ranges).

Primary Interface

  • view(): Returns a lazy range of EventWithId objects for iteration
  • viewInRange(): Returns events in a time range (requires TimeFrame for conversion)
  • addEvent()/removeEvent(): Modify events (owning storage only)
  • createView()/createFromView(): Create view/lazy backed series
  • materialize(): Convert any storage type to owning storage

Storage Backends

The class supports three storage backends (Owning, View, Lazy) that provide different performance tradeoffs. See Digital_Event_Series.hpp for details.

Example Usage


// Create and populate a series
DigitalEventSeries series;
series.setTimeFrame(my_time_frame);
series.addEvent(TimeFrameIndex{100});
series.addEvent(TimeFrameIndex{200});

// Iterate all events
for (auto event : series.view()) {
    std::cout << "Event at " << event.time().getValue() 
              << " with id " << event.id().getValue() << "\n";
}

// Query range with timeframe conversion
for (auto event : series.viewInRange(start, end, other_timeframe)) {
    process(event);
}

// Create filtered view
auto filtered = DigitalEventSeries::createView(
    std::make_shared<DigitalEventSeries>(series),
    TimeFrameIndex{50}, TimeFrameIndex{150});

Events are always sorted by time. Duplicate times are rejected.

View and Lazy storage are read-only; mutations throw std::runtime_error.

EventWithId for element accessors (time(), id(), value())

DigitalIntervalSeries for time interval data

function view

inline auto view() const

Get a std::ranges compatible view of the series.

Returns a random-access view that synthesizes EventWithId objects on demand. Uses cached pointers for fast-path iteration when storage is contiguous. Allows iterating over EventWithId objects directly.

function viewInRange

inline auto viewInRange(
    TimeFrameIndex start_index,
    TimeFrameIndex stop_index,
    TimeFrame const & source_time_frame
) const

Get events in a time range as a lazy view of EventWithId objects.

Parameters:

  • start_index Start time index (inclusive)
  • stop_index Stop time index (inclusive)
  • source_time_frame The time frame that start_index/stop_index are expressed in

Return: Lazy view of EventWithId objects in the range

Note:

  • If source_time_frame matches the series’ time frame, no conversion occurs
  • If the series has no time frame set, indices are used directly

This is the primary range query interface. It requires a source TimeFrame parameter to enable proper time conversion when the query time base differs from the series’ internal time base.
