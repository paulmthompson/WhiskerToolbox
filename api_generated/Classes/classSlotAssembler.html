Detailed Description

class SlotAssembler;

Bridge between DataManager data and model tensor I/O.

Owns a cached ModelBase instance behind a PIMPL to prevent libtorch headers from leaking into Qt translation units.

Public Types Documentation

using ProgressCallback

using SlotAssembler::ProgressCallback =  std::function<void(int current_frame, int total_frames)>;

Callback type for progress reporting during recurrent inference.

Parameters:

  • current_frame 0-based index of the frame being processed
  • total_frames Total number of frames to process

using ResultCallback

using SlotAssembler::ResultCallback =  std::function<void(std::vector<FrameResult>)>;

Callback invoked per-frame with decoded results for progressive delivery. When provided to runBatchRangeOffline(), results are pushed via this callback instead of accumulating in BatchInferenceResult::results.

using MediaOverrides

using SlotAssembler::MediaOverrides =  std::unordered_map<std::string, std::shared_ptr<MediaData> >;

Map from DataManager data_key to a cloned MediaData instance.

Used by runBatchRangeOffline() so the worker thread has its own independent MediaData (e.g. a separate VideoData with its own FFmpeg decoder state) while the UI continues using the original.

function runBatchRangeOffline

BatchInferenceResult runBatchRangeOffline(
    DataManager & dm,
    MediaOverrides const & media_overrides,
    std::vector< SlotBindingData > const & input_bindings,
    std::vector< StaticInputData > const & static_inputs,
    std::vector< OutputBindingData > const & output_bindings,
    int start_frame,
    int end_frame,
    ImageSize source_image_size,
    std::atomic< bool > const & cancel_requested,
    int batch_size =1,
    ProgressCallback const & progress =nullptr,
    ResultCallback const & result_callback =nullptr
)

Run independent inference over a frame range without writing to DataManager.

Parameters:

  • dm DataManager for non-media input encoding (masks, points, lines)
  • media_overrides Cloned MediaData instances keyed by data_key
  • input_bindings Dynamic input slot bindings
  • static_inputs Static (memory) input entries
  • output_bindings Output slot bindings
  • start_frame First frame to process (inclusive)
  • end_frame Last frame to process (inclusive)
  • source_image_size Original image dimensions
  • cancel_requested Checked before each frame; stops early if true
  • batch_size Number of frames to process per forward pass. Must be >= 1. The last chunk may contain fewer frames.
  • progress Optional callback for progress reporting
  • result_callback Optional per-frame result callback for progressive delivery. When non-null, decoded outputs are pushed via this callback and NOT accumulated in BatchInferenceResult::results.

Return: Accumulated decoded results (may be partial on cancellation). If result_callback is set, the results vector will be empty.

Identical to runBatchRange() except:

  • Uses media_overrides for MediaData lookups (thread-safe clone).
  • Returns decoded results in a BatchInferenceResult instead of calling addAtTime() on DataManager.
  • Checks cancel_requested before each frame for early exit.

function runRecurrentSequence

void runRecurrentSequence(
    DataManager & dm,
    std::vector< SlotBindingData > const & input_bindings,
    std::vector< StaticInputData > const & static_inputs,
    std::vector< OutputBindingData > const & output_bindings,
    std::vector< RecurrentBindingData > const & recurrent_bindings,
    int start_frame,
    int frame_count,
    ImageSize source_image_size,
    ProgressCallback const & progress =nullptr
)

Run sequential recurrent inference over a range of frames.

Parameters:

  • dm DataManager for encoding inputs / writing outputs
  • input_bindings Dynamic input slot bindings
  • static_inputs Static (memory) input entries
  • output_bindings Output slot bindings
  • recurrent_bindings Recurrent feedback bindings
  • start_frame First frame to process
  • frame_count Number of frames to process
  • source_image_size Original image dimensions
  • progress Optional callback for progress reporting

Exceptions:

  • std::runtime_error on inference failure

For each frame in [start_frame, start_frame + frame_count):

  1. Assemble dynamic + static inputs
  2. Inject recurrent tensors (output from previous frame) into the corresponding input slots
  3. Call forward()
  4. Decode outputs into DataManager
  5. Cache output tensors for the next frame’s recurrent inputs

Batch size is forced to 1 during recurrent inference.

bool captureStaticInput(

Capture a static input at a specific frame and store in the cache.

Parameters:

  • dm DataManager to fetch data from
  • entry Static input entry describing what to capture
  • frame The frame number to encode and cache
  • source_image_size Original image dimensions for coordinate scaling

Return: true if the tensor was successfully captured and cached

Encodes the data from DataManager at the given frame using the model’s recommended encoder for the slot, then stores the resulting tensor. The cached tensor is reused during inference for Absolute-mode entries.

function initDeviceForCurrentThread

static void initDeviceForCurrentThread()

Initialize the deep-learning device context on the calling thread.

On CUDA builds this ensures the CUDA runtime context is set for the current thread — required on Windows where CUDA per-thread state is not inherited from the main thread. Safe to call from any thread; no-op when the active device is CPU.

Call at the start of worker thread run() methods before any libtorch operations.
