Detailed Description

class KeymapSystem::KeymapManager;

Central manager for keyboard shortcut registration, resolution, and dispatch.

Responsibilities:

  • Action registry: stores all KeyActionDescriptors
  • Override management: merges defaults with user customizations
  • Conflict detection: reports same key bound to multiple actions in the same scope
  • Scope resolution: resolves key presses using priority order (EditorFocused > AlwaysRouted > Global)
  • Adapter routing: dispatches actions to registered KeyActionAdapters

When installed as a QApplication event filter, it intercepts key events and dispatches them through the scope resolution pipeline. Text-input widgets (QLineEdit, QTextEdit, etc.) are automatically bypassed.

function registerAction

bool registerAction(
    KeyActionDescriptor const & descriptor
)

Register an action descriptor.

Parameters:

  • descriptor The action metadata to register

Return: true if registered successfully, false if action_id already exists

Precondition:

  • descriptor.action_id must not be empty (enforcement: assert)
  • descriptor.scope.editor_type_id must be non-empty when scope.kind is AlwaysRouted or EditorFocused — an empty type_id is accepted silently but causes dispatchAction to match the wrong adapter class (enforcement: none)
  • descriptor.action_id must not already be registered; if it is, false is returned and the caller’s descriptor is silently ignored (enforcement: runtime_check)
  • descriptor.display_name should be non-empty for usable keybinding UI (enforcement: none)
  • descriptor.category should be non-empty for correct grouping in the keybinding editor (enforcement: none)

function cycleActiveTarget

void cycleActiveTarget(
    EditorLib::EditorTypeId const & type_id
)

Cycle the active target to the next adapter of the given type.

Parameters:

  • type_id The editor type to cycle through

When multiple adapters of the same type are registered, AlwaysRouted actions dispatch to the “active target” for that type. This method advances the active target to the next adapter in registration order, wrapping around. If no active target is set, selects the first adapter.

Emits activeTargetChanged.
