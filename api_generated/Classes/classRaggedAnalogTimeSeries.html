Detailed Description

class RaggedAnalogTimeSeries;

Ragged analog time series data structure.

RaggedAnalogTimeSeries stores time series data where each TimeFrameIndex can have a variable-length vector of float values. Unlike regular AnalogTimeSeries, the number of samples at each time point need not be constant.

This class manages:

  • Time series storage via type-erased storage wrapper (supports owning, view, and lazy)
  • TimeFrame association for time-based operations
  • Observer pattern integration for data change notifications
  • Range-based iteration interface
  • Cache optimization for fast-path iteration over contiguous storage

Storage Backends:

  • OwningRaggedAnalogStorage: Default, owns data in SoA layout
  • ViewRaggedAnalogStorage: Zero-copy filtered view of another storage
  • LazyRaggedAnalogStorage: On-demand computation from a transform view

Use cases include:

  • Spike trains with varying numbers of detected events per time bin
  • Multi-unit recordings where channel count varies over time
  • Feature vectors with time-varying dimensionality

  • The source data for a lazy view is about to be destroyed

function elements

inline auto elements() const

Get a flattened view of (TimeFrameIndex, float) pairs.

See: elementsView() for concept-compliant iteration with FlatElement

Return: A lazy range view of (TimeFrameIndex, float) pairs

This creates a lazy view that flattens the ragged structure into individual (time, value) pairs. This is the analog of RaggedTimeSeries::elements() and enables uniform iteration API across all ragged container types.

Returns std::pair<[TimeFrameIndex](/api_generated/Classes/structTimeFrameIndex.md), float> for backward compatibility with existing code that uses .first, .second, and structured bindings.

Usage: for (auto [time, value] : ragged_analog.elements()) { ... }

function elementsView

inline auto elementsView() const

Get a flattened view of FlatElement objects (concept-compliant)

See:

  • FlatElement
  • TimeSeriesConcepts.hpp for concept definitions

Return: A lazy range view of FlatElement objects

This creates a lazy view that flattens the ragged structure into individual FlatElement objects. Each element satisfies the TimeSeriesElement and ValueElement concepts, enabling use with generic time series algorithms.

Use this method when you need concept-compliant elements for generic algorithms. Use elements() when you need backward-compatible pair iteration.

Usage: ```cpp

for (auto elem : ragged_analog.elementsView()) { auto t = elem.time(); auto v = elem.value(); }



### function time_slices

inline auto time_slices() const

Get a view of (TimeFrameIndex, std::span) pairs.

Return: A lazy range view of (TimeFrameIndex, span) pairs

Similar to RaggedTimeSeries::time_slices(), this provides access to all values at each time point as a span. Useful when you need to process all values at a time together rather than individually.

Note: For lazy storage, the spans may be empty. Use time_slices_vec() instead if you need guaranteed access.
