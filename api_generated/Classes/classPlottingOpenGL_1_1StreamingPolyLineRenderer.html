Detailed Description

class PlottingOpenGL::StreamingPolyLineRenderer;

Optimized polyline renderer with incremental GPU buffer updates.

Uses glBufferSubData to update only changed portions of the vertex buffer when time series data scrolls.

function updateData

void updateData(
    CorePlotting::RenderablePolyLineBatch const & batch
)

Update buffer with new batch data using incremental strategy.

Parameters:

  • batch The new polyline batch

Precondition:

  • initialize() must have been called and returned true before the first call (enforcement: runtime_check)
  • A valid OpenGL context must be current on the calling thread (enforcement: none) [CRITICAL]
  • batch.line_start_indices.size() == batch.line_vertex_counts.size() (topology arrays must be parallel) (enforcement: none) [IMPORTANT]
  • For each i: batch.line_start_indices[i] + batch.line_vertex_counts[i] <= batch.vertices.size() / 2 (no index exceeds the vertex buffer) (enforcement: none) [CRITICAL]
  • batch.thickness > 0 (required by glLineWidth; zero or negative produces a GL error) (enforcement: none) [LOW]

Compares new data with cached data and uses glBufferSubData to update only the changed regions when possible. Falls back to full re-upload when:

  • Buffer capacity is insufficient
  • Topology changes (number of lines, line lengths)
  • First upload after clearData()

function uploadData

void uploadData(
    CorePlotting::RenderablePolyLineBatch const & batch
)

Force a full re-upload of the buffer (bypasses incremental logic)

Precondition:

  • initialize() must have been called and returned true before the first call (enforcement: runtime_check)
  • A valid OpenGL context must be current on the calling thread (enforcement: none) [CRITICAL]
  • batch.line_start_indices.size() == batch.line_vertex_counts.size() (topology arrays must be parallel) (enforcement: none) [IMPORTANT]
  • For each i: batch.line_start_indices[i] + batch.line_vertex_counts[i] <= batch.vertices.size() / 2 (no index exceeds the vertex buffer; checked against the cached batch used by render()) (enforcement: none) [CRITICAL]
  • batch.thickness > 0 (required by glLineWidth in render(); zero or negative produces a GL error) (enforcement: none) [LOW]

Use this when you know the data has changed significantly, such as when loading a new dataset.

An empty batch.vertices is handled gracefully: the buffer is cleared and the function returns without touching the GPU.
