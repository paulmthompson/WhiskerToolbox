Detailed Description

class TensorData;

Refactored N-dimensional tensor with named axes, multiple storage backends, and first-class DataManager integration.

Quick Start

// 2D time-series matrix (e.g., spectrogram)
auto spec = TensorData::createTimeSeries2D(
    flat_magnitudes, num_time_bins, num_freq_bins,
    time_storage, time_frame,
    {"0-10 Hz", "10-20 Hz", "20-30 Hz"});

auto band = spec.getColumn("10-20 Hz");

// Direct Armadillo access for mlpack
arma::fmat const& m = spec.asArmadilloMatrix();

tensor_data_refactor_proposal.md for full design rationale

function appendRow

void appendRow(
    std::span< float const > row_data
)

Append a row to the end of the tensor.

Parameters:

  • row_data Values for the new row (size must equal numColumns())

Exceptions:

  • std::logic_error if the tensor is not 2D, is backed by an unsupported storage type, or has TimeFrameIndex rows
  • std::invalid_argument if row_data.size() != numColumns()

Works for 2D tensors backed by ArmadilloTensorStorage or DenseTensorStorage with Ordinal or Interval row semantics.

For Ordinal rows, the ordinal count is incremented. For Interval rows, the provided interval is appended to the row descriptor. TimeFrameIndex rows are not supported (time index storage is immutable).

Updates the dimension descriptor (axis 0 size) and notifies observers.

function upsertRows

void upsertRows(
    std::vector< std::pair< int, std::vector< float > > > const & frame_rows,
    std::shared_ptr< TimeFrame > time_frame
)

Upsert rows keyed by TimeFrameIndex.

Parameters:

  • frame_rows Pairs of (frame_index, row_data); row_data sizes must all equal numColumns() (or each other for empty tensors)
  • time_frame Shared TimeFrame for the resulting time-indexed rows

Exceptions:

  • std::invalid_argument if frame_rows is empty, row sizes are inconsistent, or time_frame is null
  • std::logic_error if the tensor is non-empty and not 2D, or has unsupported storage

Precondition: All row_data vectors must have the same size

Merges the given frame→row-data pairs into this tensor. For each pair:

  • If a row for that frame already exists, its data is overwritten.
  • If no row exists for that frame, a new row is inserted in sorted order.

After the operation, the tensor’s rows are sorted by TimeFrameIndex and backed by a fresh TimeIndexStorage. The tensor must be 2D and backed by ArmadilloTensorStorage or DenseTensorStorage. If the tensor is empty (no storage), it is initialized from the provided rows.

Create a 2D time-series tensor (rows = time, columns = features)

Parameters:

  • data Flat row-major data (size must == num_rows * num_cols)
  • num_rows Number of rows (time points)
  • num_cols Number of columns (features / frequency bins / channels)
  • time_storage TimeIndexStorage mapping row indices to TimeFrameIndex
  • time_frame Shared TimeFrame for absolute time reference (may be nullptr; DataManager assigns the correct TimeFrame when registered)
  • column_names Optional column labels (size must match num_cols if provided)

Exceptions:

  • std::invalid_argument on size mismatches or null time_storage

Precondition:

  • time_storage != nullptr (enforcement: exception)
  • time_storage->size() == num_rows (enforcement: exception)
  • data.size() == num_rows * num_cols (enforcement: exception)
  • column_names.empty() || column_names.size() == num_cols (enforcement: exception)
  • num_rows * num_cols does not overflow std::size_t (enforcement: none) [LOW]

Uses ArmadilloTensorStorage internally. Axes named “time” and “channel”.

Create a 2D tensor with lazily-computed columns.

Parameters:

  • num_rows Number of rows (each provider must return this many floats)
  • columns Column sources (name + provider); must not be empty
  • rows Row descriptor (ordinal, time-indexed, or interval)
  • wiring Optional callback for observer invalidation wiring; called once after construction with references to the LazyColumnTensorStorage and TensorData

Exceptions:

  • std::invalid_argument if num_rows == 0 or columns is empty

Each column is backed by a type-erased provider function that computes the column data on demand. Results are cached per-column and can be invalidated to trigger recomputation.

Column names are taken from the ColumnSource::name fields. Row semantics are specified by the RowDescriptor.
